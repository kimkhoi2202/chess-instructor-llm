How to read the balanced score, honestly

The moat we fine-tune IN (tier-appropriate move selection) is where OURS-v2 leads the whole field (53% tier-fit). Its raw balanced rank is mid-pack because (1) raw fabrication is 30%, but the deployed verify-and-regenerate gate drives user-visible fabrication to ≈0%, so it is table-stakes here and down-weighted; and (2) a 1.7B model is simply less instructive than giants. The balanced winner (GLM-5) and the pragmatic base pick (Qwen3-32B) differ: tier-appropriateness is what we ADD by fine-tuning, while capacity + faithfulness + local-runnability are what a base must bring.

Faithfulness is verifier-guaranteed

The grounded fabrication rate above is the raw model. The deployed app runs a verify-and-regenerate gate: re-sample the answer up to 4× and keep the first one that passes a deterministic board-fact verifier; if none pass, fall back to an engine-derived explanation that is true by construction. So user-visible fabrication is ≈0: measured 50% → 0% on the grounded-fabrication cases tested. Sources: src/api/server.py, src/engine/faithfulness.py, VERIFIER_EVAL.md.
